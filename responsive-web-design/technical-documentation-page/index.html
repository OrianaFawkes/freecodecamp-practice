S.O.L.

Some Outerspace Language (S.O.L.) is a simple imperative programming language patterned to Java. The language makes use of references about astronomical words such as Big Bang, Orbits, Supernova, etc. In partial fulfillment of the requirements for my Compiler Design course, my group and I made this language along with its compiler written in Java.

Language Design

Character Set and Case Sensitivity

The character set makes use of the English alphabet, digits, and some special character symbols and the language is case sensitive.

Statements

  1. Input Statements

    Input statements are obtained from the users via command line, the reserved word scan is used to ask for the input.

    create string str1 = scan;
  2. Output Statements

    To display the output, the reserved word flare is used, followed by the variable name of what the user wants to display.

    create string str1 = "hello";
    flare str1;
  3. Assignment Statements

    In order to assign values to a variable, the right side of the equation involves create which indicates the creation of the variable, followed by the variable type. Then, the assignment operator denoted by the single equal sign = followed by the value of the variable name.

    create integer age = 18;
  4. Conditional Statements

    To implement if/else statements, it starts with a neutronStar which is the if statement followed by blackHole which if the condition is met, the program will run as needed. Decisions are made if and only if the pre-stated conditions are either true or false.

    create integer age = 20;
    supernova {
      neutronStar (age > 18) {
        flare "Age is greater than 18.";
      }
      blackHole {
        flare "Age is less than 18.";
      }
    }
  5. Looping Constructs

    S.O.L. supports the looping constructs for loop and while-loop. orbitFor is the indicator for for loops and orbitWhile is the indicator for the while loop.

    orbitFor (create integer i = 0; i < 10; ++i) {
      flare i;
    }
  6. Concatenations

    To concatenate variables such as integers and strings, the tilde ~ symbol is used.

    create string string1 = "Hello";
    create string string2 = "!";
    flare string1 ~ string2;

Programming Language Limitations

S.O.L. is a simple imperative programming language and is presented with some limitations:

  • No "break from loop" expression
  • No character data type
  • No modulo support
  • No float data type, the data types for numeric values are limited to integers only
  • No "else if" statement included for the conditional statements
  • No multi-line comments

Parser

Parsing Method

The parsing method used in this project is the bottom up parser LALR(1).

Parser Features

  1. When the parser encounters an error, it halts and then it outputs the line number and column number of the error
  2. The parser is able to detect the following errors:
    1. It is able to detect any unexpected tokens
    2. It is able to detect if the variable used has not yet been declared
    3. It is able to detect if a variable is being declared again
    4. When assigning a value to a variable, it can detect whether the type of the value that is going to be assigned matches the type of the variable
Interpreter

Semantic Rules checked by Interpreter

  1. Declaring an identifier again
  2. Accessing an identifier that hasn't been declared yet
  3. Mismatched types in assignment

Run-time Checking

  1. Division by zero
  2. Integer number out of range
  3. Mismatched types in input
  4. Assigning an identifier that hasn't been initialized with a value yet
    1. integer num1 = num1

Sample Codes

Multiplication Table written in SOL

bigBang;
  integer product = 0;
  integer x = scan;
  integer y = scan;
  orbitFor (integer i=1; i<=x; i++){
    product = 0;
    orbitFor (integer j=1; j<=y; j++){
      orbitFor (integer k=0; k<i; k++){
        product++;
      }
      flare toString(product)~"\t";
    }
    flare "\n";
  }
heatDeath;

GCD Calculator written in SOL

bigBang;
integer num1 = scan;
integer num2 = scan;
integer gcd = 0;
supernova{
  neutronStar(num1 == 0){
    gcd = num2;
  }
  blackHole{
    orbitWhile(num2 != 0){
      supernova{
        neutronStar(num1 > num2){
          num1 = num1-num2;
        }
        blackHole{
          num2 = num2-num1;
        }
      }
    }
    gcd = num1;
  }
}
flare "\nGCD: "~toString(gcd);
heatDeath;
